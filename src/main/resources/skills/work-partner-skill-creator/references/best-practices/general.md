## 核心工作方法论

### 目标管理：OKR 框架
- **O（Objective）**：定性的、鼓舞人心的目标，回答"我们要去哪里"
- **KR（Key Results）**：可量化的结果指标，回答"怎么衡量是否到达"
- OKR 通常按季度设定，每月复盘进展
- 区分"产出（Output）"和"结果（Outcome）"：做了多少事 vs 达到了什么效果

### 问题分析框架

**结构化思维（金字塔原理）**
```
结论
├── 论点 1 + 支撑数据
├── 论点 2 + 支撑数据
└── 论点 3 + 支撑数据
```

**SCQA 叙事结构**（用于汇报和提案）
- **S（Situation）**：背景，当前状态
- **C（Complication）**：冲突，问题或挑战
- **Q（Question）**：核心问题
- **A（Answer）**：解决方案或结论

**根因分析**
- 鱼骨图（因果图）：系统性梳理可能的原因维度
- 5Why 分析：对问题连续追问 5 次"为什么"，深挖根本原因

## 日常工作节奏

### 任务管理方法

**四象限优先级法**
| | 紧急 | 不紧急 |
|--|------|--------|
| **重要** | 立即做（危机处理） | 计划做（核心工作） |
| **不重要** | 委托或快速处理 | 减少或消除 |

**日常工作建议**
- 每天开始前：明确今日 Top 3 优先任务
- 每周开始前：回顾上周进展，规划本周重点
- 每月末：复盘 OKR 进展，调整资源分配

### 跨部门协作规范
- **需求对齐**：确认清楚"要什么"和"为什么要"，避免执行层面返工
- **里程碑管理**：大项目拆分为可检查的阶段性节点
- **沟通留痕**：重要决策通过邮件确认，减少口头承诺的风险

## 汇报与沟通

### 向上汇报原则

**结论先行**
```
1. 结论/建议（一句话）
2. 支撑数据或论据（3 条以内）
3. 所需资源或决策（如果需要）
```

**汇报常见问题**
- ❌ 叙述过程，让领导自己判断
- ❌ 数据罗列，没有提炼重点
- ✅ 先给结论，再用数据支撑
- ✅ 主动提出建议，而不只是描述问题

### 邮件写作规范
- **主题行**：清晰说明邮件目的（`【请审批】...` / `【请反馈】...` / `【知会】...`）
- **正文结构**：背景 → 核心信息 → 所需行动 → 截止时间
- **长度控制**：核心内容在前 5 行内，详细附件可附后
- **抄送规范**：只抄送真正需要知道的人

## 文档与知识管理

### 文档写作原则
- 目的明确：写给谁看？他们需要做什么决定或行动？
- 结构清晰：用标题和分段帮助读者快速定位
- 数据支撑：核心观点附上数据或案例
- 版本管理：重要文档注明版本号和最后更新日期

### 知识沉淀习惯
- 项目结束后做复盘：成功了做了什么？失败了哪里出了问题？
- 建立个人 SOP 库：重复性工作标准化，减少每次重新思考的成本
- 定期整理笔记：月度整理，确保知识可被检索

## 职业发展参考

### 核心能力模型
| 能力维度 | 说明 |
|----------|------|
| 专业技能 | 岗位核心业务能力，持续深化 |
| 数据思维 | 用数据说话，量化工作价值 |
| 沟通协作 | 向上、向下、跨部门的有效沟通 |
| 项目管理 | 目标拆解、进度管控、风险预判 |
| 学习能力 | 快速获取新知识，适应变化 |

### 工作复盘方法
每个重要项目或阶段结束后：
1. **结果**：达到目标了吗？与预期差多少？
2. **过程**：哪些决策是对的？哪些可以做得更好？
3. **学到了什么**：可复用的经验和方法
4. **下次会怎么做**：具体的行为改变计划
